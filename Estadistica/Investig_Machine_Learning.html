1. Introducción al M-L. Tipos de aprendizaje¶

Machine Learning es el área de estudio de la ciencia de la computación que analiza y construye algoritmos que manipulan data y aprenden de esta para generar modelos de pronóstico. Involucrado en áreas de estudio como análisis de datos, big data, data mining entre otros, machine learning tiene una amplia gama de aplicaciones estadísticas en rubros de suma importancia.

Existen dos factores que categorizan un método de M-L. El primero es el estímulo que puede (o no) recibir el programa luego de finalizar un ciclo de trabajo al generar un output. El otro es la presencia de información modelo o esperada que, de estar presente o no, influye en el tipo de aprendizaje al que se somete. A partir de estos dos parámetros podemos clasificar su estudio en 3 tipos de aprendizaje.

I. Aprendizaje no supervisado¶

Es aquél que se desarrolla solo a partir de la información dada inicialmente y sin ningún estímulo específico.

II. Aprendizaje por reforzamiento¶

Es aquí donde se involucra el feedback durante el proceso de aprendizaje. El algoritmo aprende qué camino tomar gracias a la constante recepción de estímulos positivos o negativos que lo entrenan a decidir qué acción previa a la recepción del estímulo influyó más en recibirlo, aspirando a optimizar su función

III. Aprendizaje supervisado¶

Tipo de apredizaje que, desde el comienzo de su ejecución, cuenta con la información a trabajar y su output esperado. El trabajo de estos algoritmos es encontrar la regla de mapeo de los campos independientes para así ser capaces de pronosticar un output de un input no presente en los datos iniciales (pronosticar resultados).

2. Métodos basados en regresión¶

Los algoritmos de regresión pertenecen al tercer tipo de aprendizaje: supervisado. En general los programas encargados en resolver problemas de regresión modelan la relación entre variables independientes y su output esperado. Acabado el modelamiento deberíamos ser capaces de pronosticar la salida de un conjunto de datos independientes no presentes al inicio del entrenamiento. Para este modelo se asume desde el inicio del proceso el tipo de relación entre los artibutos.

2.1 Algoritmos basados en regresión¶

2.1.1. Regresión lineal simple¶

Este tipo de algoritmo propio de los métodos de regresión es buen primer paso para entender en que se basan el resto. Imaginemos la dependencia más básica de variables numéricas $X$ e $Y$, donde los valores de $Y$ dependen de $X$ mediante una función afín de la forma $Y = mX + b$.

En aprendizaje supervisado, un conjunto de entrenamiento es una colección de datos previamente proporcionado que se utiliza para generar un modelo.

A continuación definiremos los elementos necesarios para el análisis de lo que llamaremos la función costo (Se utilizará la notación de Andrew Ng solo para homogeneizar la información, ver referencias)

2.1.2 Definición¶

La función de hipótesis $\hat{f} = h_{\theta}(x)$ es aquella que, dado un conjunto de datos de entrenamiento $(X,Y)$, mapea los valores independientes $X$ a un valor estimado $\hat{Y}$, tratando de acercarase el valor real $Y$.

Donde:

  • ${\theta}$: Coeficientes de ajuste o parámetros

Para el caso de regresión lineal simple, los coeficientes de la función lineal de ajuste cumplirán:

$\hat{y} = \theta_{0} + \theta_{1}x$
Luego, la función de hipótesis queda expresada como:

$h_{\theta}(x) = \theta_{0} + \theta_{1}x$

2.1.3 Definición¶

Sea el valor hipotético obtenido para un dato i-ésimo $x_i$, denotado por $h_{\theta}(x_i)$. El costo que generan la hipótesis del dato y el valor real correspondiente al mismo se define como la función:

$costo(h_{\theta}(x_i), y_i)$

La idea detrás de la implementación del costo es la de informar al algoritmo en cada iteración que, de retornarnos una estimación alejada de la realidad, el costo de este dato eleve el costo total, corrigiendo así futuras estimaciones. Esta idea se verá mejor reflejada durante el desarrollo de este tópico. Para la regresión lineal simple, el costo generado entre una hipótesis i-ésima y el valor real dado se computará como la diferencia entre los valores hipotético y real, al cuadrado.

$costo(h_{\theta}(x_i), y_i) = (h_{\theta}(x_i) - y_i)^2$


La razón detrás de usar los cuadrados de las diferencias y no las diferencias simples se verá más adelante al cubrir el concepto de gradiente descendente.

2.1.4 Definición¶

La función costo se define matemáticamente como:

$J(\theta_{0}, \theta_{1}) = \frac{1}{n}\sum_{i = 1}^{n}costo(h_{\theta}(x_i), y_i)$

Donde:

  • $\theta_{0}, \theta_{1}$: coeficientes de la función de ajuste
  • $n$: Cantidad de datos de entrenamiento
  • $costo(h(x_i), y_i)$: Función costo para el dato i-ésimo

Luego, para el caso lineal simple, la función costo quedará expresada como:

$J(\theta_{0}, \theta_{1}) = \frac{1}{n}\sum_{i = 1}^{n}(h_{\theta}(x_i) - y_i)^2$
Habiendo definido la función costo como es debido, la tarea del algoritmo regresivo será minimizar su valor en cada iteración de aprendizaje; actualizando los valores de $\theta$ para que la función afín final que generan sea cada vez mas precisa. Lograremos esto utilizando las propiedades diferenciales de una función. Sabemos que el gradiente de una función es aquél vector de derivadas parciales orientado en dirección al crecimiento más rápido. Ahora, tomemos la función costo para regresión lineal simple y calculemos su gradiente:

$\Large{\nabla J_\theta} = \large (\frac{\partial J(\theta_{0}, \theta_{1})}{\partial (\theta_{0})};\frac{\partial J(\theta_{0}, \theta_{1})}{\partial (\theta_{1})})$

Ahora bien, si un punto se desplaza sobre la superficie que genera la función costo, con el gradiente como su vector de dirección, eventualmente llegará a un máximo local; pero si tomáramos el negativo del gradiente, nos desplazaríamos a lo largo de la misma superficie en dirección a un mínimo local, minimizando la función costo para un par de coeficientes $(\theta_0, \theta_1)$.

2.1.5 Definición¶

La técnica algorítmica de optimización denominada gradiente descendente es aquella que, valiéndose de la propiedad decrementadora del negativo del gradiente de una relación, busca un valor mínimo local en una cantidad de iteraciones proporcional al valor de la derivada parcial correspondiente.

Lo que la definición nos explica es que, para minimizar una relación dada mediante la técnica, solo hará falta actualizar el valor de un parámetro (en este caso, un coeficiente de ajuste $\theta_i$) en un múltiplo del negativo del gradiente.

$valor_i \leftarrow valor_i + k(-\Large\frac{\partial f(v_1, v_2, ..., v_n)}{\partial v_i})$

El inconveniente que podemos tener es que la función derivada posea muchos minimos locales, y acabemos calculando un mínimo que posee un valor no admisible para el modelo final. Este inconveniente se resuelve gracias al grado del elemento en la sumatoria de la función costo:


$(h_{\theta}(x_i) - y_i)^2$

Si graficamos la función costo nos daremos cuenta que este elemento cuadrado dota de convexidad a la relación, brindándole un valor mínimo único.

Entonces, para la regresión lineal simple:

$\large{\nabla J(\theta_0, \theta_1)}\normalsize = \large(\frac{\partial (\frac{1}{n}\sum_{i = 1}^{n}(h_{\theta}(x_i) - y_i)^2)}{\partial(\theta_0)}; \frac{\partial (\frac{1}{n}\sum_{i = 1}^{n}(h_{\theta}(x_i) - y_i)^2)}{\partial(\theta_1)})$


Como $h_{\theta}(x_i) = \theta_0 + \theta_1 x_i$, entonces:

$ \normalsize{\nabla J(\theta_0, \theta_1)} \normalsize= (\frac{2}{n}\sum_{i = 1}^{n}(\theta_0 + \theta_1 x_i - y_i);\frac{2}{n}\sum_{i = 1}^{n}(\theta_0 + \theta_1 x_i - y_i)x_i)$

El factor $k$ presente en la asignación de la definición 1.5 también es de suma importancia, dado que de éste dependerá que el ajuste converja.

2.1.6 Definición¶

El factor de aprendizaje $\alpha$ es el valor que cuantiza en qué proporción se ajustan los parámetros involucrados por iteración ejecutada.

Es de este factor del que depende la convergencia del modelo; por lo que es vital inicializarlo con un valor adecuado. Si su valor es muy pequeño el modelo demorará en generarse, lo que resultaría en un problema de tiempo de procesamiento para una cantidad de datos significativa; en cambio, si su valor es muy grande, el error aumentaría en cada iteración y nunca se lograría generar el modelo. Para solucionar este inconveniente ajustaremos el factor de convergencia durante la iteracion para hacerla mas fina si es que tiende a diverger.

# Regresion lineal simple
# Costo
costo <- function(theta_0, theta_1, x, y){
  vector_cost <- (theta_0 + theta_1*x - y)^2
  return(vector_cost)
}

# Factor de la funcion costo derivada
costo.deriv <- function(theta_0, theta_1, x, y){
  vector_costo.deriv <- (theta_0 + theta_1*x - y)
  return(vector_costo.deriv)
}

# Funcion costo
J <- function(theta_0, theta_1, x , y){
  n <- length(x)
  return(1/n * sum(costo(theta_0, theta_1, x, y)))
}

# Gradiente de la funcion costo
J.deriv <- function(theta_0, theta_1, x , y){
  n <- length(x)
  return (c(2/n * sum(costo.deriv(theta_0, theta_1, x, y)), 2/n * sum(costo.deriv(theta_0, theta_1, x, y)*x)))
}

# Inicializamos los coeficientes a ajustar:
t0 <- 0.0
t1 <- 0.0
new_t0 <- t0
new_t1 <- t1

# Inicializamos el factor de aprendizaje
learning_factor <- 10 # Alpha

# Inicializamos los errores
error_actual <- sum(costo(t0, t1, x, y))
error_antiguo <- 0

# Captamos los datos
x <- lin_regression_data$x
y <- lin_regression_data$y

# El siguiente bucle computara hasta que el nuevo error no implique una mejora
# tan significativa en comparacion al anterior, para un valor de permisividad (0.007)

while(abs(error_actual - error_antiguo) > 0.007){
  grad_costo <- J.deriv(t0, t1, x, y)
  new_t0 <- t0 - learning_factor*grad_costo[1]
  new_t1 <- t1 - learning_factor*grad_costo[2]
  
  if(J(new_t0, new_t1, x, y) > J(t0, t1, x, y)){ # Si la regresion esta divergiendo
    learning_factor <- learning_factor/2
    next()
  }
  
  t0 <- new_t0
  t1 <- new_t1
  error_antiguo <- error_actual
  error_actual <- sum(costo(t0, t1, x, y))
}

Ahora, sea $X_i$ el vector de valores del dato i-ésimo (o fila i-ésima del data frame), y $m$ la cantidad de parametros independientes (o cantidad de columnas en el data frame). Luego, las definiciones anteriores se modifican a:

  • De la def. 2.1.2:

    $\hat{y} = h_{\theta}(X_i)= \theta_{0} + \theta_{1}x_{i1} + \theta_{2}x_{i2} + \theta_{3}x_{i3} + ... + \theta_{m}x_{im}$

    Luego:
    $h_{\theta}(X_i) = <(\theta_0, \theta_1, \theta_2, ..., \theta_n) ; (1, x_{i1}, x_{i2}, ..., x_{im})> = \theta^T X_i = X_{i}^T \theta$

  • De la def. 2.1.3:

    $costo(h_{\theta}(X_i), y_i) = (h_{\theta}(X_i) - y_i)^2 = (\theta^T X_i - y_i)^2$

  • De la def. 2.1.4:

    $J(\theta) = \frac{1}{n} \sum_{i = 1}^{n}costo(h_{\theta}(X_i), y_i) = \frac{1}{n} \sum_{i = 1}^{n}(\theta^T X_i - y_i)^2$

  • De la def. 2.1.5:

    $\large\frac{\partial J(\theta)}{\partial \theta_j} \normalsize= \frac{2}{n}\sum_{i = 1}^{n}(\theta^T X_i - y_i)x_{ij}, \small \forall j \in {0, 1, ..., m}$

# Funcion hipotesis:
h <- function(Theta, X_i){
  return(as.numeric(Theta%*%X_i))
}

# Funcion costo:
# X_i : Vector fila i-esima de datos
# y_i: valor esperado de la fila i-esima
costo <- function(Theta, X_i, y_i){
  vector_costo <- (h(Theta, X_i) - y_i)^2
  return(vector_costo)
}

# Factor de la funcion costo derivada
# X_i : Vector fila i-esima de datos
# y_i: valor esperado de la fila i-esima
costo.deriv <- function(Theta, X_i, y_i){
  vector_costo.deriv <- (h(Theta, X_i) - y_i)
  return(vector_costo.deriv)
}

# X_matrix : Matriz de valores 'x'
# Y: vector de valores esperados 'y'
J <- function(Theta, X_matrix , Y){
  n <- nrow(X_matrix)
  result <- 0
  for(i in 1:n){
    result <- result + sum(costo(Theta, X_i = X_matrix[i, ], y_i = Y[i]))
  }
  return(1/n * result)
}

# X_matrix : Matriz de valores de X
# Y: Vector de valores esperados 'y'
# j: componente a derivar parcialmente
J.deriv <- function(Theta, X_matrix , Y, j){
  n <-nrow(X_matrix)
  result <- 0
  for(i in 1:n){
    result <- result + 2/n * sum(costo.deriv(Theta, X_matrix[i, ], Y[i]))*X_matrix[i, j + 1]
  }
  return(result)
}

    n <- length(x)
    return(1/n * sum(costo(theta, x, y)))
}

# Gradiente de la funcion costo
J.grad <- function(theta, x, y){
    n <- length(x)
    part_0 <- sum(costo.deriv(theta, x, y))
    part_1 <- sum(costo.deriv(theta, x, y)*x)
    part_2 <- sum(costo.deriv(theta, x, y)*x^2)
    return(2/n * c(part_0, part_1, part_2))
}

2.1.4. Regresión logística¶

El siguiente método, si bien su nombre podría hacernos pensar que se trata de un proceso regresivo, no es así. La regresión logística es un método de clasificación que modela variables independientes a valores binarios dependientes (0 o 1). Mediante este proceso podremos generar un modelo que pronostique la probabilidad de que el ouput de una operacion lineal sea 1, ajustandose a una función que en términos probabilísticos sería equivalente a:

$P(Y = 1 \mid X)$: Probabilidad de que la salida sea 1, dado un valor de X

La necesidad de definir este tipo de clasificación (no regresión) nace debido al error drástico que surge al intentar ajustar valores con output binario mediante regresión lineal; debido a que cuando se desea pronosticar la probabilidad de un nuevo dato no presente en la fase de entrenamiento, pueden surgir salidas con valores no admisibles.

Es así como nace la idea de usar la función logística, en lugar de la función lineal, para ajustar a una curva que brinde información útil y permita generar estimaciones para valores hipotéticos.

2.1.7. Definición¶

Sea $P$ la probabilidad de que un evento $E$ ocurra; entonces se define la posibilidad de dicho evento como:

$odd = \large \frac{P}{1-P}$

Entonces si asignamos a una variable $z$ el valor de:

$z = f(y) = \normalsize ln(\frac{P(y = 1)}{1 - P(y = 1)})$

Y calculamos su inversa, el resultado será:

$P(y = 1 \mid z) = \large \frac {1}{1 + e^{-z}}$

Debido a que $y$ es una varibale binaria, estudiemos por separado los casos para $y = 0$ y $y = 1$. Sean las funciones:

$f_{y = 0}(x) = -ln(1 - h_{\theta}(x))$

knn_function <- function(point, k) {
    knn_list <- list(NULL, NULL) # Lista de k-vecinos mas cercanos
    names(knn_list) <- c("x", "y")
    for(i in 1:nrow(knn_df)) {
        test_point <- c(knn_df$x[i], knn_df$y[i])
        if(length(knn_list$x) < k) { # Si aun no hay k puntos en la lista resultante
            # Agregamos el punto actual
            knn_list$x <- c(knn_list$x, test_point[1])
            knn_list$y <- c(knn_list$y, test_point[2])
        } else { # Buscamos el punto a mayor distancia y lo reemplazamos si es viable
            biggest_dist <- -Inf
            biggest_dist_index <- 1
            for(j in 1:k){
                temp_dist <- dist(c(knn_list$x[j], knn_list$y[j]), point)
                if(temp_dist > biggest_dist){
                    biggest_dist <- temp_dist
                    biggest_dist_index <- j
                }
            }

            # Acabado el anterior 'for' tendremos el indice del punto cuya dist. a 'point' es mayor
            if(biggest_dist > dist(point, test_point)){
                knn_list$x[biggest_dist_index] <- test_point[1]
                knn_list$y[biggest_dist_index] <- test_point[2]
            }
        }
    }
    return(knn_list)
}

3.3 LVQ (Learning vector quantization)¶

Para el siguiente algoritmo es bueno tener un contexto, por lo menos general, de qué es una red neuronal.

3.1 Definición¶

Una red neuronal es un sistema de procesamiento distribuido y paralelo conformado por unidades simples de procesamiento denominadas neuronas, las cuales son capaces de almacenar información y usar el conocimiento aprendido.

Sus principales propiedades son:

  • 1. El conocimiento es obtenido durante una fase de aprendizaje (entrenamiento)
  • 2. La fuerza de interconexión entre neuronas, conocida como peso sináptico es usado para almacenar el conocimiento

Luego, el procedimiento algorítmico en la fase de aprendizaje consistirá en modificar dichos pesos actualizando su valor a fin de obtener un modelo. Ahora bien, en un algoritmo LVQ aplicamos esta teoría para utilizar las neuronas como herramientas de clasificación, modificando sus pesos a fin de que al finalizar su entrenamiento, su información sea suficiente para representar una única clase. Veremos en su implementación que el algoritmo presenta la misma idea de minimizar el costo de un valor para beneficiar el aprendizaje, actualizando los parámetros de clasificación (en este caso, los pesos de cada neurona) en un múltiplo de un vector de variación.

En el ejemplo, tomamos como datos de entrenamiento a la información de las columnas 1, 3 y 5 del dataframe iris, proveniente del paquete estándar de $R$. Estas columnas corresponden a los atributos Sepal.Length, Petal.Length y Species, y será ésta última la que modelaremos. Además, generamos las tres neuronas correspondientes a las clases (Species) existentes, con valores iniciales tomados aleatoriamente de la data inicial. Inicializamos el factor de aprendizaje $\alpha$ ya conocido en 0.1 (por regla el factor debe empezar con un valor en $<0,1>$) y ploteamos.

Podemos observar como las 3 formas geométricas correspondientes a las 3 clases que pretendemos predecir se posicionan de manera aleatoria sobre algún punto dado. Se buscará actualizar las posiciones de estas neuronas de tal manera que al finalizar el modelo, sirvan de centros para la clasificación de datos no presentes. Geométricamente, observaremos como las neuronas se acercarán cada vez más en cada iteración a su clase correspondiente, finalizando en una posición central por cada clase (coloreada en el ploteo). Análogo al caso de regresión, buscaremos minimizar el costo (calculado como la distancia euclidiana) entre un dato cualquiera y una neurona cercana pertinente.

El algoritmo funciona así:

  • 0. Inicializamos el factor de aprendizaje ($\alpha$) tal que 0 < $\alpha$ < 1
  • 1. Para cada dato $x_i$ buscamos la neurona más cercana con vector peso $w_j$
  • 2. Si esta neurona corresponde a la misma clase del dato, la neurona se acerca al dato, caso contrario se aleja
  • 3. Repetir los pasos 1 y 2 para todos los datos de entrenamiento
  • 4. Disminuir el factor de aprendizaje $\alpha$ (de manera proporcional)
  • 5. Repetir los pasos 1 $\rightarrow$ 4 bajo una condición de convergencia.

Lo extraño de este algoritmo es que es difícil delimitar esta última condición. Según la bibliografía, se utiliza un número de iteraciones fijo o se revisa que $\alpha$ sea lo suficientemente pequeño.

Para simplificar la implementación nos referiremos equivalentemente a las neuronas y sus pesos, que solo serán los campos numéricos de cada neurona correspondientes al ajuste de Sepal.Lenght y Petal.Length.

El criterio de actualización de las neuronas, como mencionamos anteriormente, consiste en acercar una neurona a un dato de la misma clase, y alejarla de datos de clase diferente. Para un dato $x_i$ supongamos que la neurona (refiriéndonos al peso de esta) $w_i$ posee en su campo Species un valor igual al del dato; entonces, en un siguiente instante, los nuevos valores de los campos de $w_i$ se calcularán como:

$w_i^{t+1} \leftarrow w_i^{t} + \alpha (x_i - w_i^{t})$, con *t* referente a un instante de tiempo

Caso contrario, si $x_i$ y $w_i$ poseen valores distintos en su campo Species, entonces $w_i$ se actualiza como:

$w_i^{t+1} \leftarrow w_i^{t} - \alpha (x_i - w_i^{t})$, con *t* referente a un instante de tiempo

Notemos cómo se asegura la convergencia al aumentar o decrementar en un valor de $\pm \alpha (x_i - w_i^{t})$. Dado que la norma de este vector es justo la distancia entre el dato y el peso de la neurona correspondiente, y el factor $\alpha$ lo normaliza, nunca nos excederemos del dato al acercarnos o alejarnos.

Entonces la implementación del algoritmo es el siguiente:

    new_neuron.coord <- NULL
    
    # Actualizacion de la neurona (peso de la neurona)
    if(current_data$Species == current_winner$Species){
      new_neuron.coord <- as.vector(current_winner$Coord.) + learning_factor*desp_vector
    } else {
      new_neuron.coord <- as.vector(current_winner$Coord.) - learning_factor*desp_vector
    }
    
    neurons_df$Sepal.Length[current_winner$index] <- new_neuron.coord[1]
    neurons_df$Petal.Length[current_winner$index] <- new_neuron.coord[2]
  }
  # Ajuste de ALPHA  
  learning_factor <- learning_factor/10
}

4. Algoritmos basados en árboles¶

Aún dentro del problema de clasificación, muchas veces es útil dar resultados de los pronósticos como algún tipo de receta de pasos útil y fácilmente empleable para quien los necesite. Otra necesidad que surge es la de explorar los datos de entrenamiento particionando sus dominios en valores, lo que hace que el barrido sobre la información sea logarítmicamente más rápido. Éstas y otras muchas ventajas nos la proporciona la implementación de modelos supervisados sobre algoritmos basados en árboles.

4.1. Definición¶

En un problema de clasificación para $C$ clases, un límite de decisión (decision boundary) es aquel hiperplano que limita instancialmente al conjunto de trabajo (la mayoría de veces siendo un espacio vectorial) en $C$ conjuntos disjuntos

Por ejemplo, de los algoritmos basados en distancia y para tres clases ($C = 3$) dependientes de sólo dos atributos independientes, gráficamente el límite de decision está representado como curvas o polígonos (abiertos o cerrados) para cada clase.

Veremos que un algoritmo basado en árboles particionará el dominio de los datos de entrenamiento de una manera distinta a esta, debido a su naturaleza recursiva.

La idea general de éstos algoritmos (ya sea para aplicar clasificación o regresión) es la de generar nodos a lo largo de la altura de su estructura que pongan a prueba binariamente los atributos independientes del dato (esto, claro está, durante la fase de entrenamiento), y dependiendo de la salida de estas pruebas ir modelando el dato hasta que en una hoja obtengamos el valor hipotético clasificatorio.

4.2. Definición informal¶

Para un nodo asociado a un parámetro, se define su grado de homogeneida como la cantidad de datos que convergen a un solo tipo de resultado categórico (clase).

Con intención de aclarar estas ideas, veamos la siguiente imagen en la que generamos los sub-árboles correspondientes a los nodos de los parámetros independientes.

Como vemos, el atributo salarios posee 3 datos en casos homogéneos (celdas de verde), deudas no los presenta y, finalmente, existe uno para el número de familiares; por lo tanto, los grados de homogeneidad son 3, 0 y 1 respectivamente. Ahora bien, aquél atributo con mayor grado aparecerá primero en el árbol de clasificación (o regresión), debido a que posicionarlo primero clasificará mas rápido los datos de manera absoluta y disminuirá la altura del árbol (disminuyendo a su vez el costo). Es importante optimizar su altura dado que al minimizarla disminuimos la cantidad de pruebas a realizar. Por ejemplo, puede que obtener el estado de deudas de un usuario que pide un préstamo implique gastos elevados para el banco, y preguntarle al mismo cuántos familiares tiene no le cueste nada en lo absoluto, por lo que al banco le conviene no tener que recurrir a averiguar el estado de deudas si es que con la información brindada por la cantidad de familiares basta.

Aún existen dos inconvenientes. El primero es que, para el ejemplo esto puede funcionar, pero para una gran cantidad de datos (numéricos o no) será casi imposible obtener el grado de homogeneidad para un atributo, dado que este siempre sería cero. Sin embargo existen funciones basadas en teoría de la información que son capaces de medir la homogeneidad de los datos utilizando entropía. La otra dificultad es que aún necesitamos que estos subárboles se dividan binariamente, por lo que emplearemos otra función que particione recursivamente los datos.

Resolviendo el segundo inconveniente tenemos algunas ideas para implementar el critero de partición de nodos:

Criterios¶

Sea un nodo $A$ correspondiente a alguna propiedad, $P(A)$ la función de probabilidad de $A$ y $R(A)$ el riesgo de $A.$ Se generarán los nodos $A_L$ y $A_R$ izquierdo y derecho respectivamente según el siguiente criterio

$P(A_L)R(A_L) + P(A_R)R(A_R) \leq P(A)R(A)$

Si bien la matemática es un poco abstracta y en realidad no sabemos la manera en la que se generan los nodos hasta que se cumpla esta desigualdad, esta es suficiente condición para que recursivamente se generen los nodos hijos de cualquier nodo correspondiente a los atributos independientes. Lo que buscamos es generar un par de nodos hijos de tal manera que $\Delta R$ se maximice; es decir, exista el mayor decremento de riesgo. Este criterio no es bueno del todo debido a que la generación de los nodos hijos queda fijada tan pronto como se cumpla la desigualdad, pero podrían haber otras generaciones que optimicen el $\Delta R.$

Entonces, usando el criterio anterior de medir la homogeneidad de los subarboles de atributos, pero ahora de manera más general, usando una función $f$ que opera la impuridad (la no-homogeneidad) del atributo podemos definir:

$I(A) = f(p_{iA})$


Donde:
$\normalsize A$: Nodo
$\normalsize p_{iA}$: Proporción de los datos de la propiedad $A$ que pertenecen a la clase $i$ en un futuro (estimación)

Luego, de manera equivalente a como hacíamos en los algoritmos de regresión, minimizaremos el costo de generar el modelo, a partir de la variación de la impuridad, tal que:

$\Delta I = p(A)I(A) - p(A_L)I(A_L) - p(A_R)I(A_R)$


Esta será la función que usemos al implementar los CART's.

Continuaremos definiendo algunos tópicos de la teoría de la información para desarrollar este nuevo método.

4.3 Definición¶

En la teoría de la información, se define a la Entropía como la medida del desorden (incertidumbre) en una muestra de datos $S$ dada.

$H(S) = \displaystyle \sum_{x \in X} p(x)ln(\frac{1}{p(x)})$

Donde:

  • $S$: Conjunto de datos pertenecientes a un atributo (columna en un data frame)
  • $X$: Conjunto de clases de la data total
  • $p(x)$: Proporción de datos con clase igual a $x$ en función a la cantidad de datos total en $S$

Note la analogía con el grado de homogeneidad. En este caso la entropía calcula de manera más general barriendo cada uno de los parámetros $S$ independientes presentes en el conjunto de datos. Dado que esta función calcula el grado de desorden para cada atributo y hemos revisado anteriormente que lo que deseamos es empezar a clasificar la data comenzando con el atributo más homogéneo, es natural que pensemos en iniciar la clasificación en este nuevo algoritmo con los campos de menor entropía como nodos padre, pero se sabe (de la teoría de la información) que no basta con el valor entrópico de un atributo para fijar un criterio de división, debido a que existen muchas maneras de dividir los valores del atributo para continuar la clasificación. (Revisar la desigualdad del apartado de criterios de este capítulo). Para asegurar la fidelidad del resultado definimos:

4.4 Definición¶

La ganancia de información $IG(A)$ es la medida de qué tanta información se pudo captar a partir un conjunto de datos $S$ al dividirlo en uno de sus atributos $A$.

$IG(A,S) = H(S) - \displaystyle \sum_{t \in T} p(t)H(t)$

Donde los términos son equivalentes a la definición de entropía y:

  • $T$: Subconjuntos generados al dividir $S$ en función al parámetro $A$ ($S = \bigcup\limits_{t \in T} t$)

Con esta nueva función ya podemos asegurar que la división de los parámetros, según los subconjuntos generados, es adecuada sólo con maximizar la ganancia de información durante el proceso. Gracias a la ganancia de información disminuiremos la incertidumbre al generar los nodos hijos, dado que el atributo que genere mayor ganancia será utilizado para continuar la división del árbol, y debido a que se realiza iterativamente, el criterio de división siempre será el adecuado (único).

Algoritmo¶

  • 1 Calcular la entropía para todos los datos del conjunto $S$
  • 2 Calcular el dato A con mayor IG del conjunto
  • 3 Generar un árbol a partir del nodo contiendo a A
  • 4 Evaluar las ramas generadas en esta división
    • 4.1 Si una rama presenta solo un tipo de atributo de clasficación, convertir la rama en hoja con dicho atributo
    • 4.2 Si una rama no presenta datos ajustables, convertir rama en hoja con un atributo igual al de mayor frecuencia en el nodo A (padre)
    • 4.3 Caso contrario, aplicar el algoritmo en la rama resultante

5. Algoritmos Bayesianos¶

Existen otras maneras de clasificar datos que son más eficientes en comparación a los algoritmos que hemos revisado previamente; esto siempre que se cumplan algunas condiciones iniciales.

Recordando la teoría probabilística involucrada en la Regla de Bayes, tenemos que para un vector de datos $(x_1, x_2, ..., x_n)$ y una clase $C$ se cumple:

$P(C\mid (x_1, x_2, ..., x_n)) = \large\frac{P((x_1, x_2, ..., x_n)\mid C)P(C)}{P(x_1, x_2, ..., x_n)}$

La forma de interpretar esto en un contexto de clasificación (que es lo que nos compete actualmente) es de la siguiente manera:

  • $P(C\mid (x_1, x_2, ..., x_n))$: Probabilidad de que, según los valores dados en el dato $x$, se clasifique este en la clase $C$

  • $P((x_1, x_2, ..., x_n)\mid C)$: Probabilidad de que, dada una clase $C$, el valor $x$ pertenezca a esta

  • $P(C)$: Probabilidad de que cualquier dato arbitrario pertenezca a $C$

  • $P(x_1, x_2, ..., x_n)$: Probabilidad del dato $X$

Esto se puede considerar un abuso de notación dado que estamos calculando la probabilidad de un vector de $n$ componentes condicionándola con una única clase $C$. La manera correcta de expresarlo sería:

$P(x_1\mid C; x_2\mid C; ...; x_n\mid C)$

Donde la probabilidad se aplica a todos los atributos del dato vectorial $x$. Veremos como utilizar este resultado para clasificar los datos según las asumpciones que hagamos sobre el conjunto de entrenamiento.

5.1 Naïve Bayes¶

El primer estilo de clasificación utilizando la regla de bayes es un acercamiento muy inocente y simple (haciendo alusión a su nombre), asumiendo que la data $x$ ingresada presenta valores en sus componentes que son independientes entre sí; por lo tanto la ecuación vista se transforma en:


$P(x_1\mid C; x_2\mid C; ...; x_n\mid C) = \frac{\displaystyle \prod_{i = 1}^{n} [P(x_i\mid C)].P(C)}{\Large P(x_1, x_2, ..., x_n)}$

Donde:

  • $P(C)$: Probabilidad total de la clase
  • $P(x_1, ..., x_n)$: Probabilidad del dato
  • $P(x_i\mid C)$: Probabilidad del atributo i-esimo dado que se tiene en una clase $C$

Como ejemplo podríamos introducir unos cuántos valores explicativos. Sean los datos de reventa de una concesionaria:

Luego, si quisiésemos clasificar un nuevo automóvil usado con los datos $x$ = (19, B) recurriríamos a la fórmula:

  • Asumiendo que SI se pudiese poner en reventa: $$ \begin{aligned} P(SI \mid (19, B)) &= \frac{P((19, B) \mid SI)P(SI)}{P(19,B)} \\ &=\frac{P(19 \mid SI).P(B \mid SI).P(SI)}{P(19).P(B)} \\ &=\frac{(2/4).(1/4).(4/6)}{(2/6).(3/6)} = 0.5 \end{aligned} $$
  • Asumiendo que NO se pudiese poner en reventa: $$ \begin{aligned} P(NO \mid (19, B)) &= \frac{P((19, B) \mid NO)P(NO)}{P(19,B)} \\ &=\frac{P(19 \mid NO).P(B \mid NO).P(NO)}{P(19).P(B)} \\ \end{aligned} $$

Como $P(19 \mid NO) = 0$, entonces:

  • $P(SI \mid (19, B))$ = .5
  • $P(NO \mid (19, B))$ = 0

Como .5 > 0, la clasificación resultante asignaría al dato $x$ = (19, B) como APTO (SI) a reventa. Revisando el código correspondiente, para una base de datos aleatoria, tenemos:

# Funcion: prob_cond (probabilidad condicional)
# Inputs:
#   A_i: Atributo i-esimo (valor de la columna i-esima del dato ingresado)
#   i: indice de ubicacion del atributo i-ésimo (numero de la columna)
#   C: Valor de la clase de supuesta pertenencia
# La data está almacenada en un df global cuya ultima columna atribuye a las clases
prob_cond <- function(A_i, i, C){
    temp_dataf <- test_data[, c(i, ncol(test_data))] # Obtenemos las columnas de interes
    temp_dataf <- temp_dataf[temp_dataf[ncol(temp_dataf)] == C,]
    n <- nrow(temp_dataf)
    temp_dataf <- temp_dataf[temp_dataf[, 1] == A_i, ]
    return(nrow(temp_dataf)/n)
}

# Funcion: prob_class (probabilidad de la clase)
# Input: La clase a revisar
# Output: Probabilidad total de la clase
prob_class <- function(C){
    temp_df <- test_data[test_data[ncol(test_data)] == C, ]
    return (nrow(temp_df)/nrow(test_data))
}

# Funcion: prob_class (probabilidad de la clase)
# Input: Dato A_i a revisar y su ubicacion en el df con indice 'i'
# Output: Probabilidad total del dato
prob_value <- function(A_i, i){
    temp_df <- test_data[test_data[i] == A_i, ]
    return(nrow(temp_df)/nrow(test_data))
}

# Funcion regla de Bayes
# Input: Dato a clasificar con componentes (A_1, A_2, ..., A_n, C = Supuesta clase)
# Output: Probabilidad condicional de dicho dato
regla_bayes <- function(x){
    indep_values <- x[c(1:(length(x) - 1))]
    product_1 <- 1 # Producto del numerador
    product_2 <- 2 # Producto del denominador
    for(i in 1:length(indep_values)){
        product_1 <- product_1*prob_cond(indep_values[i], i, x[length(x)])
        product_2 <- product_2*prob_value(indep_values[i], i)
    }
    product_1 <- product_1*prob_class(x[length(x)])
    result <- product_1/product_2
    if(is.null(result) || is.nan(result)) 
        return (0)
    return (result)
}

Explicando un poco la función class_naive.bayes, lo que se hace es ingresar un dato a clasificar en la función y barrer todas las clases disponibles en la data, calculando la probabilidad de que dicho dato pertenezca a la clase adjuntada por iteración. Note que la complejidad del algoritmo depende de la cantidad de clases presentes, por lo que el método de naïve bayes es viable para una gran cantidad de datos siempre que la cantidad de clases existentes sea pequeña.

6. Algoritmos de agrupación¶

El siguiente tópico desarrolla un nuevo tipo de aprendizaje muy útil para la clasificación (sobre todo para una cantidad considerablemente grande de data). Muchas veces el conjunto de datos que requerimos clasificar no indica la clase a la que pertenece, por lo que el conjunto de entrenamiento no determina ni la cantidad de clases a considerar ni sus posible valores. Los algoritmos de clusterización pertenecen al grupo de aprendizaje no supervisado, mediante los cuales se clasificará data sin saber con antelación las posibles clases a las que pueden pertenecer cada entrada. La idea general en este nuevo tipo de aprendizaje es intentar generar un dato central que represente cada clase, por lo que necesitaremos alguna función que centralice este dato durante la fase de aprendizaje.

6.1 Definición¶

En un problema de agrupación, un clúster se define como la representación de un subgrupo del total de datos observados de tal manera que elementos pertenecientes a un mismo clúster presentan algún tipo de similitud cuantizable.

6.1 K-means (K-medias)¶

Dado que la media de un conjunto de valores se define estadísticamente como una medida de tendencia central, podemos utilizarla como regularizador durante la fase de entrenamiento.

6.2 Definición¶

En un problema de clusterización, se define al centroide como un punto representativo de una clase o clúster.

Se detalla el algoritmo a continuación:

  • Mientras exista algún dato que cambie de clúster respecto al paso anterior:
    • Para todos los datos:
      • 1. Asignarle al dato el centroide más cercano para incluirlo en su clúster respectivo
    • Para todos los centroides:
      • 1. Calcular el vector de medias de los datos que pertenezcan a su clúster
      • 2. Cambiar el valor de su posición a la media previamente calculada

# Distancia euclideana
euclid_dist <- function(v1, v2){
  v <- v1 - v2
  return(sqrt(sum(v^2)))
}

# get_near.centroid
# input: vector de dos componentes
# output: indice del centroide mas cercano
get_near.centroid <- function(data_index){
  current_dist <- Inf
  near.centroid_index <- NULL
  point <- c(clust_data$x[data_index], clust_data$y[data_index])
  for(i in 1:nrow(centroids_df)){
    new_centroid <- c(centroids_df$x[i], centroids_df$y[i])
    if(euclid_dist(new_centroid, point) < current_dist){
      near.centroid_index <- i
      current_dist <- euclid_dist(new_centroid, point)
    }
  }
  return(near.centroid_index)
}

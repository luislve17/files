1. Introducción al M-L. Tipos de aprendizaje¶

Machine Learning es el rea de estudio de la ciencia de la computacin que analiza y construye algoritmos que manipulan data y aprenden de esta para generar modelos de pronstico. Involucrado en reas de estudio como anlisis de datos, big data, data mining entre otros, machine learning tiene una amplia gama de aplicaciones estadsticas en rubros de suma importancia.

Existen dos factores que categorizan un mtodo de M-L. El primero es el estmulo que puede (o no) recibir el programa luego de finalizar un ciclo de trabajo al generar un output. El otro es la presencia de informacin modelo o esperada que, de estar presente o no, influye en el tipo de aprendizaje al que se somete. A partir de estos dos parmetros podemos clasificar su estudio en 3 tipos de aprendizaje.

I. Aprendizaje no supervisado¶

Es aqul que se desarrolla solo a partir de la informacin dada inicialmente y sin ningn estmulo especfico.

II. Aprendizaje por reforzamiento¶

Es aqu donde se involucra el feedback durante el proceso de aprendizaje. El algoritmo aprende qu camino tomar gracias a la constante recepcin de estmulos positivos o negativos que lo entrenan a decidir qu accin previa a la recepcin del estmulo influy ms en recibirlo, aspirando a optimizar su funcin

III. Aprendizaje supervisado¶

Tipo de apredizaje que, desde el comienzo de su ejecucin, cuenta con la informacin a trabajar y su output esperado. El trabajo de estos algoritmos es encontrar la regla de mapeo de los campos independientes para as ser capaces de pronosticar un output de un input no presente en los datos iniciales (pronosticar resultados).

2. Métodos basados en regresión¶

Los algoritmos de regresin pertenecen al tercer tipo de aprendizaje: supervisado. En general los programas encargados en resolver problemas de regresin modelan la relacin entre variables independientes y su output esperado. Acabado el modelamiento deberamos ser capaces de pronosticar la salida de un conjunto de datos independientes no presentes al inicio del entrenamiento. Para este modelo se asume desde el inicio del proceso el tipo de relacin entre los artibutos.

2.1.2 Definición¶

La funcin de hiptesis $\hat{f} = h_{\theta}(x)$ es aquella que, dado un conjunto de datos de entrenamiento $(X,Y)$, mapea los valores independientes $X$ a un valor estimado $\hat{Y}$, tratando de acercarase el valor real $Y$.

Donde:

  • ${\theta}$: Coeficientes de ajuste o parmetros

Para el caso de regresin lineal simple, los coeficientes de la funcin lineal de ajuste cumplirn:

$\hat{y} = \theta_{0} + \theta_{1}x$
Luego, la funcin de hiptesis queda expresada como:

$h_{\theta}(x) = \theta_{0} + \theta_{1}x$

2.1.3 Definición¶

Sea el valor hipottico obtenido para un dato i-simo $x_i$, denotado por $h_{\theta}(x_i)$. El costo que generan la hiptesis del dato y el valor real correspondiente al mismo se define como la funcin:

$costo(h_{\theta}(x_i), y_i)$

La idea detrs de la implementacin del costo es la de informar al algoritmo en cada iteracin que, de retornarnos una estimacin alejada de la realidad, el costo de este dato eleve el costo total, corrigiendo as futuras estimaciones. Esta idea se ver mejor reflejada durante el desarrollo de este tpico. Para la regresin lineal simple, el costo generado entre una hiptesis i-sima y el valor real dado se computar como la diferencia entre los valores hipottico y real, al cuadrado.

$costo(h_{\theta}(x_i), y_i) = (h_{\theta}(x_i) - y_i)^2$


La razn detrs de usar los cuadrados de las diferencias y no las diferencias simples se ver ms adelante al cubrir el concepto de gradiente descendente.

2.1.4 Definición¶

La funcin costo se define matemticamente como:

$J(\theta_{0}, \theta_{1}) = \frac{1}{n}\sum_{i = 1}^{n}costo(h_{\theta}(x_i), y_i)$

Donde:

  • $\theta_{0}, \theta_{1}$: coeficientes de la funcin de ajuste
  • $n$: Cantidad de datos de entrenamiento
  • $costo(h(x_i), y_i)$: Funcin costo para el dato i-simo

Luego, para el caso lineal simple, la funcin costo quedar expresada como:

$J(\theta_{0}, \theta_{1}) = \frac{1}{n}\sum_{i = 1}^{n}(h_{\theta}(x_i) - y_i)^2$
Habiendo definido la funcin costo como es debido, la tarea del algoritmo regresivo ser minimizar su valor en cada iteracin de aprendizaje; actualizando los valores de $\theta$ para que la funcin afn final que generan sea cada vez mas precisa. Lograremos esto utilizando las propiedades diferenciales de una funcin. Sabemos que el gradiente de una funcin es aqul vector de derivadas parciales orientado en direccin al crecimiento ms rpido. Ahora, tomemos la funcin costo para regresin lineal simple y calculemos su gradiente:

$\Large{\nabla J_\theta} = \large (\frac{\partial J(\theta_{0}, \theta_{1})}{\partial (\theta_{0})};\frac{\partial J(\theta_{0}, \theta_{1})}{\partial (\theta_{1})})$

Ahora bien, si un punto se desplaza sobre la superficie que genera la funcin costo, con el gradiente como su vector de direccin, eventualmente llegar a un mximo local; pero si tomramos el negativo del gradiente, nos desplazaramos a lo largo de la misma superficie en direccin a un mnimo local, minimizando la funcin costo para un par de coeficientes $(\theta_0, \theta_1)$.

2.1.5 Definición¶

La tcnica algortmica de optimizacin denominada gradiente descendente es aquella que, valindose de la propiedad decrementadora del negativo del gradiente de una relacin, busca un valor mnimo local en una cantidad de iteraciones proporcional al valor de la derivada parcial correspondiente.

Lo que la definicin nos explica es que, para minimizar una relacin dada mediante la tcnica, solo har falta actualizar el valor de un parmetro (en este caso, un coeficiente de ajuste $\theta_i$) en un mltiplo del negativo del gradiente.

$valor_i \leftarrow valor_i + k(-\Large\frac{\partial f(v_1, v_2, ..., v_n)}{\partial v_i})$

El inconveniente que podemos tener es que la funcin derivada posea muchos minimos locales, y acabemos calculando un mnimo que posee un valor no admisible para el modelo final. Este inconveniente se resuelve gracias al grado del elemento en la sumatoria de la funcin costo:


$(h_{\theta}(x_i) - y_i)^2$

Si graficamos la funcin costo nos daremos cuenta que este elemento cuadrado dota de convexidad a la relacin, brindndole un valor mnimo nico.

Entonces, para la regresin lineal simple:

$\large{\nabla J(\theta_0, \theta_1)}\normalsize = \large(\frac{\partial (\frac{1}{n}\sum_{i = 1}^{n}(h_{\theta}(x_i) - y_i)^2)}{\partial(\theta_0)}; \frac{\partial (\frac{1}{n}\sum_{i = 1}^{n}(h_{\theta}(x_i) - y_i)^2)}{\partial(\theta_1)})$


Como $h_{\theta}(x_i) = \theta_0 + \theta_1 x_i$, entonces:

$ \normalsize{\nabla J(\theta_0, \theta_1)} \normalsize= (\frac{2}{n}\sum_{i = 1}^{n}(\theta_0 + \theta_1 x_i - y_i);\frac{2}{n}\sum_{i = 1}^{n}(\theta_0 + \theta_1 x_i - y_i)x_i)$

El factor $k$ presente en la asignacin de la definicin 1.5 tambin es de suma importancia, dado que de ste depender que el ajuste converja.

2.1.6 Definición¶

El factor de aprendizaje $\alpha$ es el valor que cuantiza en qu proporcin se ajustan los parmetros involucrados por iteracin ejecutada.

Es de este factor del que depende la convergencia del modelo; por lo que es vital inicializarlo con un valor adecuado. Si su valor es muy pequeo el modelo demorar en generarse, lo que resultara en un problema de tiempo de procesamiento para una cantidad de datos significativa; en cambio, si su valor es muy grande, el error aumentara en cada iteracin y nunca se lograra generar el modelo. Para solucionar este inconveniente ajustaremos el factor de convergencia durante la iteracion para hacerla mas fina si es que tiende a diverger.

costo <- function(theta_0, theta_1, x, y){
  vector_cost <- (theta_0 + theta_1*x - y)^2
  return(vector_cost)
}

costo.deriv <- function(theta_0, theta_1, x, y){
  vector_costo.deriv <- (theta_0 + theta_1*x - y)
  return(vector_costo.deriv)
}

# Funcion costo
J <- function(theta_0, theta_1, x , y){
  n <- length(x)
  return(1/n * sum(costo(theta_0, theta_1, x, y)))
}

# Gradiente de la funcion costo
J.deriv <- function(theta_0, theta_1, x , y){
  n <- length(x)
  return (c(2/n * sum(costo.deriv(theta_0, theta_1, x, y)), 2/n * sum(costo.deriv(theta_0, theta_1, x, y)*x)))
}

J <- function(Theta, X_matrix , Y){
  n <- nrow(X_matrix)
  result <- 0
  for(i in 1:n){
    result <- result + sum(costo(Theta, X_i = X_matrix[i, ], y_i = Y[i]))
  }
  return(1/n * result)
}

  }

2.1.4. Regresión logística¶

El siguiente mtodo, si bien su nombre podra hacernos pensar que se trata de un proceso regresivo, no es as. La regresin logstica es un mtodo de clasificacin que modela variables independientes a valores binarios dependientes (0 o 1). Mediante este proceso podremos generar un modelo que pronostique la probabilidad de que el ouput de una operacion lineal sea 1, ajustandose a una funcin que en trminos probabilsticos sera equivalente a:

$P(Y = 1 \mid X)$: Probabilidad de que la salida sea 1, dado un valor de X

La necesidad de definir este tipo de clasificacin (no regresin) nace debido al error drstico que surge al intentar ajustar valores con output binario mediante regresin lineal; debido a que cuando se desea pronosticar la probabilidad de un nuevo dato no presente en la fase de entrenamiento, pueden surgir salidas con valores no admisibles.

Es as como nace la idea de usar la funcin logstica, en lugar de la funcin lineal, para ajustar a una curva que brinde informacin til y permita generar estimaciones para valores hipotticos.

2.1.7. Definición¶

Sea $P$ la probabilidad de que un evento $E$ ocurra; entonces se define la posibilidad de dicho evento como:

$odd = \large \frac{P}{1-P}$

Entonces si asignamos a una variable $z$ el valor de:

$z = f(y) = \normalsize ln(\frac{P(y = 1)}{1 - P(y = 1)})$

Y calculamos su inversa, el resultado ser:

$P(y = 1 \mid z) = \large \frac {1}{1 + e^{-z}}$

3.3 LVQ (Learning vector quantization)¶

Para el siguiente algoritmo es bueno tener un contexto, por lo menos general, de qu es una red neuronal.

3.1 Definición¶

Una red neuronal es un sistema de procesamiento distribuido y paralelo conformado por unidades simples de procesamiento denominadas neuronas, las cuales son capaces de almacenar informacin y usar el conocimiento aprendido.

Sus principales propiedades son:

  • 1. El conocimiento es obtenido durante una fase de aprendizaje (entrenamiento)
  • 2. La fuerza de interconexin entre neuronas, conocida como peso sinptico es usado para almacenar el conocimiento

Luego, el procedimiento algortmico en la fase de aprendizaje consistir en modificar dichos pesos actualizando su valor a fin de obtener un modelo. Ahora bien, en un algoritmo LVQ aplicamos esta teora para utilizar las neuronas como herramientas de clasificacin, modificando sus pesos a fin de que al finalizar su entrenamiento, su informacin sea suficiente para representar una nica clase. Veremos en su implementacin que el algoritmo presenta la misma idea de minimizar el costo de un valor para beneficiar el aprendizaje, actualizando los parmetros de clasificacin (en este caso, los pesos de cada neurona) en un mltiplo de un vector de variacin.

Anlogo al caso de regresin, buscaremos minimizar el costo (calculado como la distancia euclidiana) entre un dato cualquiera y una neurona cercana pertinente.

El algoritmo funciona as:

  • 0. Inicializamos el factor de aprendizaje ($\alpha$) tal que 0 < $\alpha$ < 1
  • 1. Para cada dato $x_i$ buscamos la neurona ms cercana con vector peso $w_j$
  • 2. Si esta neurona corresponde a la misma clase del dato, la neurona se acerca al dato, caso contrario se aleja
  • 3. Repetir los pasos 1 y 2 para todos los datos de entrenamiento
  • 4. Disminuir el factor de aprendizaje $\alpha$ (de manera proporcional)
  • 5. Repetir los pasos 1 $\rightarrow$ 4 bajo una condicin de convergencia.

Lo extrao de este algoritmo es que es difcil delimitar esta ltima condicin. Segn la bibliografa, se utiliza un nmero de iteraciones fijo o se revisa que $\alpha$ sea lo suficientemente pequeo.

Para simplificar la implementacin nos referiremos equivalentemente a las neuronas y sus pesos, que solo sern los campos numricos de cada neurona correspondientes al ajuste de Sepal.Lenght y Petal.Length.

El criterio de actualizacin de las neuronas, como mencionamos anteriormente, consiste en acercar una neurona a un dato de la misma clase, y alejarla de datos de clase diferente. Para un dato $x_i$ supongamos que la neurona (refirindonos al peso de esta) $w_i$ posee en su campo Species un valor igual al del dato; entonces, en un siguiente instante, los nuevos valores de los campos de $w_i$ se calcularn como:

$w_i^{t+1} \leftarrow w_i^{t} + \alpha (x_i - w_i^{t})$, con *t* referente a un instante de tiempo

Caso contrario, si $x_i$ y $w_i$ poseen valores distintos en su campo Species, entonces $w_i$ se actualiza como:

$w_i^{t+1} \leftarrow w_i^{t} - \alpha (x_i - w_i^{t})$, con *t* referente a un instante de tiempo

Notemos cmo se asegura la convergencia al aumentar o decrementar en un valor de $\pm \alpha (x_i - w_i^{t})$. Dado que la norma de este vector es justo la distancia entre el dato y el peso de la neurona correspondiente, y el factor $\alpha$ lo normaliza, nunca nos excederemos del dato al acercarnos o alejarnos.

Criterios¶

Sea un nodo $A$ correspondiente a alguna propiedad, $P(A)$ la funcin de probabilidad de $A$ y $R(A)$ el riesgo de $A.$ Se generarn los nodos $A_L$ y $A_R$ izquierdo y derecho respectivamente segn el siguiente criterio

$P(A_L)R(A_L) + P(A_R)R(A_R) \leq P(A)R(A)$

Si bien la matemtica es un poco abstracta y en realidad no sabemos la manera en la que se generan los nodos hasta que se cumpla esta desigualdad, esta es suficiente condicin para que recursivamente se generen los nodos hijos de cualquier nodo correspondiente a los atributos independientes. Lo que buscamos es generar un par de nodos hijos de tal manera que $\Delta R$ se maximice; es decir, exista el mayor decremento de riesgo. Este criterio no es bueno del todo debido a que la generacin de los nodos hijos queda fijada tan pronto como se cumpla la desigualdad, pero podran haber otras generaciones que optimicen el $\Delta R.$

Entonces, usando el criterio anterior de medir la homogeneidad de los subarboles de atributos, pero ahora de manera ms general, usando una funcin $f$ que opera la impuridad (la no-homogeneidad) del atributo podemos definir:

$I(A) = f(p_{iA})$


Donde:
$\normalsize A$: Nodo
$\normalsize p_{iA}$: Proporcin de los datos de la propiedad $A$ que pertenecen a la clase $i$ en un futuro (estimacin)

Luego, de manera equivalente a como hacamos en los algoritmos de regresin, minimizaremos el costo de generar el modelo, a partir de la variacin de la impuridad, tal que:

$\Delta I = p(A)I(A) - p(A_L)I(A_L) - p(A_R)I(A_R)$


Esta ser la funcin que usemos al implementar los CART's.

4.3 Definición¶

En la teora de la informacin, se define a la Entropa como la medida del desorden (incertidumbre) en una muestra de datos $S$ dada.

$H(S) = \displaystyle \sum_{x \in X} p(x)ln(\frac{1}{p(x)})$

Donde:

  • $S$: Conjunto de datos pertenecientes a un atributo (columna en un data frame)
  • $X$: Conjunto de clases de la data total
  • $p(x)$: Proporcin de datos con clase igual a $x$ en funcin a la cantidad de datos total en $S$

Note la analoga con el grado de homogeneidad. En este caso la entropa calcula de manera ms general barriendo cada uno de los parmetros $S$ independientes presentes en el conjunto de datos. Dado que esta funcin calcula el grado de desorden para cada atributo y hemos revisado anteriormente que lo que deseamos es empezar a clasificar la data comenzando con el atributo ms homogneo, es natural que pensemos en iniciar la clasificacin en este nuevo algoritmo con los campos de menor entropa como nodos padre, pero se sabe (de la teora de la informacin) que no basta con el valor entrpico de un atributo para fijar un criterio de divisin, debido a que existen muchas maneras de dividir los valores del atributo para continuar la clasificacin. (Revisar la desigualdad del apartado de criterios de este captulo). Para asegurar la fidelidad del resultado definimos:

4.4 Definición¶

La ganancia de informacin $IG(A)$ es la medida de qu tanta informacin se pudo captar a partir un conjunto de datos $S$ al dividirlo en uno de sus atributos $A$.

$IG(A,S) = H(S) - \displaystyle \sum_{t \in T} p(t)H(t)$

Donde los trminos son equivalentes a la definicin de entropa y:

  • $T$: Subconjuntos generados al dividir $S$ en funcin al parmetro $A$ ($S = \bigcup\limits_{t \in T} t$)

Con esta nueva funcin ya podemos asegurar que la divisin de los parmetros, segn los subconjuntos generados, es adecuada slo con maximizar la ganancia de informacin durante el proceso. Gracias a la ganancia de informacin disminuiremos la incertidumbre al generar los nodos hijos, dado que el atributo que genere mayor ganancia ser utilizado para continuar la divisin del rbol, y debido a que se realiza iterativamente, el criterio de divisin siempre ser el adecuado (nico).

Algoritmo¶

  • 1 Calcular la entropa para todos los datos del conjunto $S$
  • 2 Calcular el dato A con mayor IG del conjunto
  • 3 Generar un rbol a partir del nodo contiendo a A
  • 4 Evaluar las ramas generadas en esta divisin
    • 4.1 Si una rama presenta solo un tipo de atributo de clasficacin, convertir la rama en hoja con dicho atributo
    • 4.2 Si una rama no presenta datos ajustables, convertir rama en hoja con un atributo igual al de mayor frecuencia en el nodo A (padre)
    • 4.3 Caso contrario, aplicar el algoritmo en la rama resultante

5. Algoritmos Bayesianos¶

Existen otras maneras de clasificar datos que son ms eficientes en comparacin a los algoritmos que hemos revisado previamente; esto siempre que se cumplan algunas condiciones iniciales.

Recordando la teora probabilstica involucrada en la Regla de Bayes, tenemos que para un vector de datos $(x_1, x_2, ..., x_n)$ y una clase $C$ se cumple:

$P(C\mid (x_1, x_2, ..., x_n)) = \large\frac{P((x_1, x_2, ..., x_n)\mid C)P(C)}{P(x_1, x_2, ..., x_n)}$

La forma de interpretar esto en un contexto de clasificacin (que es lo que nos compete actualmente) es de la siguiente manera:

  • $P(C\mid (x_1, x_2, ..., x_n))$: Probabilidad de que, segn los valores dados en el dato $x$, se clasifique este en la clase $C$

  • $P((x_1, x_2, ..., x_n)\mid C)$: Probabilidad de que, dada una clase $C$, el valor $x$ pertenezca a esta

  • $P(C)$: Probabilidad de que cualquier dato arbitrario pertenezca a $C$

  • $P(x_1, x_2, ..., x_n)$: Probabilidad del dato $X$

Esto se puede considerar un abuso de notacin dado que estamos calculando la probabilidad de un vector de $n$ componentes condicionndola con una nica clase $C$. La manera correcta de expresarlo sera:

$P(x_1\mid C; x_2\mid C; ...; x_n\mid C)$

Donde la probabilidad se aplica a todos los atributos del dato vectorial $x$. Veremos como utilizar este resultado para clasificar los datos segn las asumpciones que hagamos sobre el conjunto de entrenamiento.

$P(x_1\mid C; x_2\mid C; ...; x_n\mid C) = \frac{\displaystyle \prod_{i = 1}^{n} [P(x_i\mid C)].P(C)}{\Large P(x_1, x_2, ..., x_n)}$

Donde:

  • $P(C)$: Probabilidad total de la clase
  • $P(x_1, ..., x_n)$: Probabilidad del dato
  • $P(x_i\mid C)$: Probabilidad del atributo i-esimo dado que se tiene en una clase $C$

Luego, si quisisemos clasificar un nuevo automvil usado con los datos $x$ = (19, B) recurriramos a la frmula:

  • Asumiendo que SI se pudiese poner en reventa: $$ \begin{aligned} P(SI \mid (19, B)) &= \frac{P((19, B) \mid SI)P(SI)}{P(19,B)} \\ &=\frac{P(19 \mid SI).P(B \mid SI).P(SI)}{P(19).P(B)} \\ &=\frac{(2/4).(1/4).(4/6)}{(2/6).(3/6)} = 0.5 \end{aligned} $$
  • Asumiendo que NO se pudiese poner en reventa: $$ \begin{aligned} P(NO \mid (19, B)) &= \frac{P((19, B) \mid NO)P(NO)}{P(19,B)} \\ &=\frac{P(19 \mid NO).P(B \mid NO).P(NO)}{P(19).P(B)} \\ \end{aligned} $$

Como $P(19 \mid NO) = 0$, entonces:

  • $P(SI \mid (19, B))$ = .5
  • $P(NO \mid (19, B))$ = 0

Como .5 > 0, la clasificacin resultante asignara al dato $x$ = (19, B) como APTO (SI) a reventa. Revisando el cdigo correspondiente, para una base de datos aleatoria, tenemos:

# Funcion regla de Bayes
# Input: Dato a clasificar con componentes (A_1, A_2, ..., A_n, C = Supuesta clase)
# Output: Probabilidad condicional de dicho dato
regla_bayes <- function(x){
    indep_values <- x[c(1:(length(x) - 1))]
    product_1 <- 1 # Producto del numerador
    product_2 <- 2 # Producto del denominador
    for(i in 1:length(indep_values)){
        product_1 <- product_1*prob_cond(indep_values[i], i, x[length(x)])
        product_2 <- product_2*prob_value(indep_values[i], i)
    }
    product_1 <- product_1*prob_class(x[length(x)])
    result <- product_1/product_2
    if(is.null(result) || is.nan(result)) 
        return (0)
    return (result)
}

6. Algoritmos de agrupación¶

El siguiente tpico desarrolla un nuevo tipo de aprendizaje muy til para la clasificacin (sobre todo para una cantidad considerablemente grande de data). Muchas veces el conjunto de datos que requerimos clasificar no indica la clase a la que pertenece, por lo que el conjunto de entrenamiento no determina ni la cantidad de clases a considerar ni sus posible valores. Los algoritmos de clusterizacin pertenecen al grupo de aprendizaje no supervisado, mediante los cuales se clasificar data sin saber con antelacin las posibles clases a las que pueden pertenecer cada entrada. La idea general en este nuevo tipo de aprendizaje es intentar generar un dato central que represente cada clase, por lo que necesitaremos alguna funcin que centralice este dato durante la fase de aprendizaje.

6.1 Definición¶

En un problema de agrupacin, un clster se define como la representacin de un subgrupo del total de datos observados de tal manera que elementos pertenecientes a un mismo clster presentan algn tipo de similitud cuantizable.

6.1 K-means (K-medias)¶

Dado que la media de un conjunto de valores se define estadsticamente como una medida de tendencia central, podemos utilizarla como regularizador durante la fase de entrenamiento.

6.2 Definición¶

En un problema de clusterizacin, se define al centroide como un punto representativo de una clase o clster.

Se detalla el algoritmo a continuacin:

  • Mientras exista algn dato que cambie de clster respecto al paso anterior:
    • Para todos los datos:
      • 1. Asignarle al dato el centroide ms cercano para incluirlo en su clster respectivo
    • Para todos los centroides:
      • 1. Calcular el vector de medias de los datos que pertenezcan a su clster
      • 2. Cambiar el valor de su posicin a la media previamente calculada

# Distancia euclideana
euclid_dist <- function(v1, v2){
  v <- v1 - v2
  return(sqrt(sum(v^2)))
}

# get_near.centroid
# input: vector de dos componentes
# output: indice del centroide mas cercano
get_near.centroid <- function(data_index){
  current_dist <- Inf
  near.centroid_index <- NULL
  point <- c(clust_data$x[data_index], clust_data$y[data_index])
  for(i in 1:nrow(centroids_df)){
    new_centroid <- c(centroids_df$x[i], centroids_df$y[i])
    if(euclid_dist(new_centroid, point) < current_dist){
      near.centroid_index <- i
      current_dist <- euclid_dist(new_centroid, point)

  }

}
